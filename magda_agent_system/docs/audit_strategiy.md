# Отчет об аудите торговых стратегий IntelliTrader

## Обзор
Был проведен анализ C# кода в модулях `IntelliTrader.Trading` и `IntelliTrader.Rules`. Основное внимание уделялось логике трейлинга (Trailing Buy/Sell) и механизмам безопасности.

## Результаты анализа

### 1. Логика Трейлинга (Trailing Logic)
- **Файл**: `IntelliTrader.Trading/TimedTasks/TradingTimedTask.cs`
- **Реализация**: Трейлинг реализован корректно через отслеживание `BestTrailingMargin`. При достижении заданного отклонения (`Trailing`) срабатывает ордер.
- **Замечание**: Логика Buy Trailing использует `InitialPrice` для расчета маржи. Стоит рассмотреть возможность динамического обновления `InitialPrice` при сильных движениях рынка вниз, чтобы "подтягивать" точку входа.

### 2. Проверки безопасности
- **Положительные стороны**:
    - Присутствуют проверки на минимальный баланс (`CanBuy`).
    - Есть защита от "пыли" (dust) в `CanSell`.
    - Имеется механизм `TradingSuspended` для экстренной остановки.
- **Риски**:
    - В `CanBuy` проверка `BuySamePairTimeout` может блокировать усреднение (DCA), если параметры настроены слишком жестко.
    - Нет проверки на аномальный спред перед исполнением трейлинг-ордера, что может привести к покупке по невыгодной цене на неликвидных парах.

### 3. Эффективность
- Использование `ConcurrentDictionary` для активных трейлингов позволяет безопасно обрабатывать пары в многопоточной среде.
- Частота проверки регулируется через `TradingCheckInterval`.

## Рекомендации и новые задачи
1. **Добавить проверку спреда**: Внедрить проверку `MaxSpread` непосредственно перед исполнением ордера после завершения трейлинга.
2. **Оптимизация DCA**: Проверить взаимодействие `BuySamePairTimeout` и логики DCA, чтобы избежать блокировки запланированных усреднений.
3. **Автоматизация сборки**: Настроить CI/CD для создания портативных (portable) релизов.

## Новая приоритетная задача
